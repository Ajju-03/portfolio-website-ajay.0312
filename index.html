End-to-End Diabetes Risk Prediction System
End-to-end ML pipeline · SQL + Python · REST API
  • A real ML pipeline isn't just a model — it's data integrity, reproducibility, and production reliability working together.
  • This project builds a fully modular, object-oriented ML pipeline on the PIMA Diabetes dataset, covering every stage from raw data to live REST API. I benchmarked four models — Logistic Regression, Random Forest, Gradient Boosting, and SVM — using stratified k-fold cross-validation and selected Gradient Boosting based on AUC-ROC as the primary metric, achieving 79% test accuracy.
  • What sets this apart is the production layer: predictions are logged to a MySQL database via SQLAlchemy ORM, with a diagnosed and fixed data duplication bug caused by repeated append-mode inserts across API restarts — the kind of real-world data integrity issue that matters at scale. All training experiments — hyperparameters, metrics, and artifacts — are tracked with MLFlow for full reproducibility.
  • The final model is served as a FastAPI REST API with Pydantic request validation, supporting real-time single-record inference via JSON endpoints.

Transformer-Based Sentiment Analysis: From Scratch to API
Deep learning NLP model · TensorFlow · FastAPI
  • Most people fine-tune BERT. I built the Transformer itself. This project is a ground-up implementation of a Transformer encoder in TensorFlow/Keras, following the original Attention Is All You Need paper — complete with multi-head self-attention, sinusoidal positional encoding, residual connections, and layer normalisation across stacked encoder layers. No pre-built model. No shortcuts.
  • Trained on the IMDB 50K sentiment dataset with a custom tokenizer (20K vocab), stratified train/val/test splits, early stopping, learning rate scheduling, and ModelCheckpoint callbacks. The result: 87.9% test accuracy and 0.954 AUC — competitive with fine-tuned transformer baselines.
  • Every experiment was tracked end-to-end with MLFlow — hyperparameters, per-epoch metrics (accuracy, AUC, precision, recall), and model artifacts — making runs fully reproducible. The trained model was then deployed as a production FastAPI REST API, exposing /predict and /predict/batch endpoints with structured JSON responses including confidence scores and class probabilities.
